health_check

The Health_check gem provides a simple service that Pingdom, NewRelic, EngineYard, uptime.openacs.org and other uptime services can monitor to confirm the rails application is healthy, or alert you if it isn't.

The health checks include checking the database, (nfs/local) file system, email gateway, cache are accessible and that migrations are up to date. You can also add your own custom checks as well.

Installation

  • Add the line gem "health_check" to your Gemfile,
  • run bundle
  • deploy to your server
  • Configure your uptime service to monitor http://your.site/health_check for the text "success" (and alert you on failure)

health_check is compatible with rails versions 2.3 through 4.0 - See travis-ci tests for further details
